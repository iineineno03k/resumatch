# プロジェクト概要

## サービス名

**ResuMatch**（レジュマッチ）

## 背景・目的

### なぜ作るのか

- 副業で開発受託案件を獲得したい
- エージェント経由だと「本業持ち」だと案件が見つかりにくい
- Wantedlyで直接応募するも、面接プロセスが非効率で打率が低い

### 解決策

「こいつガチだな」と思わせるポートフォリオとしてSaaSを作る

### このSaaSの役割

- 実際に売上を出すことが目的ではない（出たらラッキー程度）
- 受託案件の営業時に見せる「武器」として機能させる
- 「ローカルで動くだけのポートフォリオ」ではなく、LP含めて本番デプロイされた状態にする
- クライアントに「このレベルのものを作れる人なんだ」と思わせる

## プロダクト概要

**採用管理システム（ATS: Applicant Tracking System）+ AI履歴書解析**

### 選定理由

- 「採用」は誰でも知ってる業務で説明不要
- AIの使いどころが明確（履歴書PDF → 自動解析・要約）
- スタートアップ〜中小企業どこでも使う
- 画面イメージしやすい（応募者一覧 → 詳細 → AIサマリー）

## ユースケース

### UC1: 求人ポジションを管理したい

「エンジニア」「営業」など複数ポジションの募集を管理

- 求人作成（タイトル、募集要項、必須スキル等）
- 求人一覧表示
- 求人のステータス管理（募集中/停止）

### UC2: 応募者を登録・管理したい

応募が来たら候補者として登録

- 応募者登録（名前、メール、電話、応募ポジション）
- 履歴書PDFアップロード
- 応募者一覧表示（ポジション別にフィルタ）
- 応募者詳細表示

### UC3: 履歴書をAIで解析したい（★メイン機能）

履歴書見るのダルい → AIが要点抽出

- 履歴書PDF → AIが自動解析
- スキル・経歴・資格を構造化して抽出
- AIによる候補者サマリー生成（「Java5年、PM経験あり、〜」）
- 求人要件とのマッチ度スコア（オプション）

### UC4: 選考ステータスを進めたい

書類選考→面接→内定の流れを管理

- ステータス変更（書類選考中/1次面接/2次面接/内定/不採用）
- ステータス別の応募者フィルタ

### UC5: メモ・評価を残したい

面接後のフィードバック共有

- 応募者ごとにメモ追加
- 評価（☆1〜5）
- 複数人でコメント

### UC6: ログイン・チーム管理

複数人で使う前提

- ユーザー登録・ログイン
- チーム（企業）作成
- メンバー招待

## MVP（最小構成）

デモ映えする最小構成：

1. ログイン/登録
2. 求人一覧・作成
3. 応募者一覧（リスト形式）
4. 応募者登録（PDF付き）
5. 応募者詳細 + **AI解析結果**（★ここが見せ場）

### MVP対象外（後回し）

- カンバンボード（ドラッグ&ドロップ）
- チーム招待機能
- ダッシュボード
