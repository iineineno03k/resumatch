# タスク完了時チェックリスト

## コード変更後に必ず実行

### 1. リント・フォーマット
```bash
bun run lint      # エラーがないか確認
bun run format    # フォーマット適用
```

### 2. テスト
```bash
bun run test:run  # テストが通るか確認
```

### 3. ビルド確認 (大きな変更時)
```bash
bun run build     # ビルドエラーがないか確認
```

## UI コンポーネント作成時

1. コンポーネント実装
2. Storybook ストーリー作成
3. `bun run storybook` で確認
4. ユーザーにレビュー依頼
5. OK が出たら次へ

## API 実装時

1. エンドポイント実装
2. **認証チェック** を忘れずに
3. **team_id フィルタリング** を忘れずに
4. **個人情報をログ出力しない**
5. 単体テスト作成

## AI/PDF 処理実装時

1. モジュール実装
2. **単体テストを先に書く** (必須)
3. モジュール単位で精度担保
4. 統合テスト

## コミット前

1. `bun run lint` パス
2. `bun run test:run` パス
3. コミットメッセージにプレフィックスをつける
4. 日本語で記述

## タスク管理

Phase 完了時は `/task-done` コマンドを実行して進捗を更新する。
